AI Media Monitoring

Proposed AWS Architecture for Chartered Accountants — Conduct Matter Detection

Stage 1
Scrape
Fetch content from 20+ regulatory and news sources on schedule. Lambda for public HTML, Fargate + Playwright for authenticated/JS sites.
→
Stage 2
Extract
AI · Claude via Bedrock
Extract names, actions, context, and identifying details from raw article text. Textract for PDFs (ASIC Gazettes).
→
Stage 3
Match
AI · Claude via Bedrock
Fuzzy match extracted profiles against Salesforce CRM. Compound reasoning: name + employer + location + role = confidence.
→
Stage 4
Associate
AI · Rules + LLM
Group findings into potential cases. Rule-based first pass, LLM for ambiguous associations.
→
Stage 5
Review
React SPA dashboard for conduct staff. Triage queue: confirm, dismiss, adjust, or split cases. Full audit trail in DynamoDB.

Orchestration

  • EventBridge Scheduler — daily/hourly cron triggers
  • Step Functions — pipeline orchestration, retries, parallel execution per source

Scraping

  • Lambda — ~15 public HTML sites (TPB, ATO, ASIC, APRA, etc.)
  • Fargate + Playwright — authenticated sites (AFR, Accountants Daily)
  • Secrets Manager — credential storage

AI Processing

  • Bedrock (Claude) — entity extraction, fuzzy matching, case reasoning
  • Textract — PDF text extraction (ASIC Gazettes, enforcement reports)

Storage

  • S3 — raw content, PDFs, evidence snapshots
  • DynamoDB — findings, cases, reviews, audit log

Review Interface

  • React SPA — hosted on S3 + CloudFront
  • API Gateway + Lambda — REST API for CRUD
  • Cognito — authentication for conduct staff

Notifications & Reporting

  • SNS / SES — email alerts for high-confidence matches
  • QuickSight — compliance dashboards (post-PoC)

AI Model Decisions — Where LLM Earns Its Keep (and Where It Might Not)

Extraction

Extract names, actions, and context from varied article formats — regulatory notices, court decisions, news stories.
PROPOSED LLM (Claude) — handles all formats
ALT NER (spaCy/BERT) — fast, cheap, names only
ALT Regex — works for structured sources
ALT Hybrid — regex + LLM by source type

Matching

Match extracted profiles against Salesforce CRM members using compound signals — name, employer, location, role.
PROPOSED LLM (Claude) — holistic reasoning
ALT Fuzzy strings (Jaro-Winkler)
ALT Embedding similarity (cosine)
ALT Two-stage: fuzzy filter → LLM rank

Case Association

Group findings from different sources into potential cases about the same person or matter.
PROPOSED Rules + LLM for ambiguous cases
ALT Rule-based only (name + timeframe)
ALT Embedding clustering (HDBSCAN)
ALT Graph clustering (co-occurrence)
